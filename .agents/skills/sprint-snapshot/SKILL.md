## Goal

Render the current sprintctl sprint state to a committed plaintext snapshot that serves as the shared, diffable record of sprint progress.

## Steps

1. Run `sprintctl render` to produce the current sprint document.
2. Write the output to `docs/sprint-snapshots/sprint-current.txt`.
3. Stage and commit: `git add docs/sprint-snapshots/sprint-current.txt && git commit -m "chore: update sprint snapshot"`.
4. Push immediately per project convention.

## When to use

- After marking a batch of items done or blocked.
- At sprint close before running `kctl-extract`.
- When handing off to a reviewer or another agent.

## Do not

- Do not edit the snapshot file manually — it is always generated by `sprintctl render`.
- Do not commit snapshot updates as part of feature commits; keep them as standalone `chore:` commits.
