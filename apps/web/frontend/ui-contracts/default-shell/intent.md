# Intent - Default Shell

## Direction

- Surface: `apps/web/frontend/app/`
- Audience: operator and household admin
- Delivery lane: publish
- Density: standard
- Motion: subtle

## Taste brief

Build a calm analytical shell for the primary product routes.

- Mood: warm, editorial, trustworthy
- Priorities: clear hierarchy, readable summaries, easy navigation, low visual noise
- Anti-goals: cold enterprise chrome, novelty gradients, dashboard clutter

## Priority surfaces

- dashboard
- reports
- runs
- upload
- control

## Acceptance cues

- summary cards read clearly without chart-heavy decoration
- control routes feel operational but still approachable
- the shell can host denser admin content without breaking the tone
