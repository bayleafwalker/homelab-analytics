# Intent - Retro Shell

## Direction

- Surface: `apps/web/frontend/app/retro`
- Audience: operator and analyst
- Delivery lane: publish
- Density: compact
- Motion: subtle

## Taste brief

Build a parallel CRT-inspired shell for dense operational use.

- Mood: terminal-adjacent, deliberate, control-room oriented
- Priorities: dense information display, strong hierarchy, keyboard-friendly flow
- Anti-goals: novelty glow overload, fuzzy glass panels, marketing-page layouts

## Priority surfaces

- retro overview
- retro money
- retro utilities
- retro operations
- retro control and terminal entry

## Acceptance cues

- the shell feels like an intentional alternate renderer, not a skin
- operational labels and status bands remain readable at a glance
- detail routes preserve density without collapsing into visual clutter
