Ops Console

new route: apps/web/frontend/app/console/page.js

A power-user terminal sibling to the retro shell. Single screen, 6 panels, no scrolling. Built on the same retro tokens; aimed at people who already live in retro-shell.

Build order
  • Add a <ConsoleShell> that mirrors RetroShell but locks to a 3×2 CSS Grid panel layout.
  • Each panel = <Panel title right> with a thin top header row.
  • Panels: cashflow ticker, spend heatmap (DOW × hour), attention, runs tail, category bars + ratios, command stub.
  • Wire heatmap from a new endpoint or a transform of spendByCategoryMonthly grouped by booked timestamp.
  • Keyboard: F1..F5 jump panel, ⌘K opens command.
Visual rules
  • One font: var(--retro-font-mono). One size scale: 10/11/12/14/18/38.
  • Colors only ok / warn / accent / muted; no decorative hues.
  • Numbers right-aligned, tabular-nums, no thousands separators inside panels.
  • Panel headers all caps, letter-spacing: 0.16em.
Avoid
  • CRT vignette & scanlines on top of bar charts — reserve them for the Wallboard.

Prompt

@retro-shell.js @components/sparkline-chart.js

Create a new route /console that uses a "ConsoleShell" derived from
RetroShell, but laid out as a strict CSS grid:

  grid-template-columns: 1.1fr 1.4fr 1fr;
  grid-template-rows: 1fr 1fr;

Six panels, in order:
  1. Cashflow ticker     — latest net + 12m sparkline + Δ12M %

  3. Attention           — rankedAttention list with S1/S2/S3 markers
  4. Runs tail           — getRuns(8); table with run_id/source/file/rows/status
  5. Categories + ratios — horizontal bars from getSpendByCategoryMonthly

  6. Command stub        — non-interactive scrollback of recent
                            assistant_confidence calls; bottom-line cursor

Use existing retro-* CSS variables. Do not import any new fonts.
Add hotkeys F1-F5 to focus panels via querySelector(`[data-panel-id="${n}"]`).
Add ⌘K to focus the command stub.
new component: ConsoleShell new component: HeatGrid new endpoint: getSpendHeatmap(window) a11y: keyboard map documented in panel

Morning Briefing

new route: apps/web/frontend/app/briefing/page.js (also good as the email digest)

A daily front page assembled from the same data sources. Calm, narrative, share-friendly; uses Fraunces (or Iowan) for display and Inter for body. Should also serialize cleanly to PDF/email.

Build order
  • Compose the lede string server-side from cashflow + utility + recurring totals.
  • Render the chart-of-the-week as one wide SVG with three series and an explicit caption.
  • Three columns: Decisions (top 3 attention), Trends (3 sparklines with deltas), Quiet (3 sentences from rules).
  • Add a /briefing.json response so the worker can email it daily.
Visual rules
  • Two horizontal rules only: under the masthead, above the columns.
  • Eyebrow color encodes column intent: warm = lede cool = chart ok = quiet.
  • Body type 19px / 1.55, max 52ch.
Avoid
  • Decorative imagery — the chart is the image.
  • Anything below the fold that isn't text or one chart.

Cashflow Map

new route: apps/web/frontend/app/cashflow/page.js

Sankey of where the money actually goes: income sources → allocation buckets → endpoints. Useful for sharing with a partner, and for spotting drift between months.

Build order
  • Add getCashflowSankey(month) on the API; it returns { nodes, links } with three layers (source/bucket/end).
  • Compute layout server-side — node y from cumulative share, link height from value.
  • Render with plain SVG; the prototype's SankeyLink bezier is sufficient.
  • Add a month picker; persist selection in URL ?m=2026-04.
  • Below the diagram, four KPI cards (fixed share, discretionary, saved, buffer).
Data shape
// /api/reports/cashflow-sankey?m=2026-04
{
  nodes: [
    { id: "salary", layer: "source", label: "Salary", amount: 4820 },
    { id: "fixed",  layer: "bucket", label: "Fixed costs", amount: 1793 },
    { id: "mortgage", layer: "end", label: "Mortgage", amount: 1284 }
  ],
  links: [
    { from: "salary", to: "fixed", value: 1793 },
    { from: "fixed",  to: "mortgage", value: 1284 }
  ]
}

Homelab Wallboard

new route: apps/web/frontend/app/wallboard/page.js (intended for a wall display)

An always-on status board for the homelab itself — services, hosts, devices, and a 24h power chart. It shares tokens with the retro shell and the Ops Console, but is built to be readable from across the room.

Build order
  • Wire to the existing HA bridge (test_ha_bridge* tells us the endpoints).
  • Add getServiceHealth() — returns name, host, status, uptime, cpu, mem.
  • Add getDeviceReadings() — per-room temperature, humidity, instantaneous power.
  • Add getEventFeed(since=12h) — merge ingestion runs + HA action proposals + alerts.
  • Auto-refresh every 30s with SWR or a simple interval; freeze on hover.
  • Add a ?compact=1 mode for 1080p TVs (drop the event feed, double the device tile size).
Visual rules
  • Minimum readable size = 14px; numbers 18px+; status text 24px+.
  • Use scanlines & vignette; they're the wallboard signature.
  • Colors: ok green, warn amber, info cyan. Errors red — bold, no background.
  • Keep one and only one ticker line at the bottom.
Avoid
  • Animations beyond the cursor blink and ticker scroll.
  • Mixing serif type into the wallboard — mono only.

Prompt

★

Shared groundwork

do this once; every concept benefits

Three small additions to the design system unblock all five concepts and keep the existing pages from drifting.

1. Tokens
  • Add --font-display (Fraunces or Iowan) alongside the existing sans/mono pair.
  • Add a --line-strong at rgba(19,40,51,0.28) for editorial rules.
  • Document a 6-step type scale: 11 / 13 / 15 / 19 / 28 / 56.
2. Primitives
  • <Pill tone> — replaces the ad-hoc .statusPill chains.
  • <Spark values> — lightweight wrapper around SparklineChart.
  • <NumMono> — tabular numbers, used everywhere a money value renders.
  • <Eyebrow color> — the small caps line every panel uses.
3. Motion
  • One transition only: 120ms ease on color/border/transform.
  • Wallboard cursor blink is the only timed animation.
  • Respect prefers-reduced-motion.
4. Storybook
  • Add a story per new component (Pulse, Horizon, ConsolePanel, Sankey, ServiceTile).
  • Use the existing app-shell.stories.jsx as the template.
  • Snapshot each new top-level page (operating-picture, briefing, console, cashflow, wallboard).
